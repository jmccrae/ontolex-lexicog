This document describes the lexicography module of the Lexicon Model for Ontologies (lemon) as a result of the work of the Ontology Lexica community group (OntoLex). The module is targeted at the representation of dictionaries and any other linguistic resource containing lexicographic data, and addresses structures and annotations commonly found in lexicography. This module operates in combination with the lemon core module, referred to as OntoLex.

Introduction

Background and motivation

The lemon model provides a core vocabulary (OntoLex) to represent linguistic information associated to ontology and vocabulary elements. The model follows the principle of semantics by reference in the sense that the semantics of a lexical entry is expressed by reference to an individual, class or property defined in an ontology.

The current version of lemon (as an outcome of the OntoLex group, sometimes referred as OntoLex-lemon in the literature) as well as its previous version (lemon [1]) have been increasingly used in the context of dictionaries and lexicographical data to convert existent lexicographic information into the standards and formats of the Semantic Web. Such preliminary experiences comprise monolingual [2], bilingual [3], and multilingual [4] dictionaries, as well as diachronic [5], dialectal [6], and etymological ones [7], among others. The added value of using linked data technologies in lexicography and its implications for the micro and macro structure of dictionaries have been explored as well by several authors [8, 9].

After analysing the literature, the proposers of this module perceived an obvious need for reaching some agreement that allows for a better and more inter-operable migration of existing dictionaries into linked data [10]. For illustration, the OGL ontology [11] has its own notion of dictionary entry materialised in the ogl:Entry class, while in [4] the ad-hoc kd:dictionaryEntry relation was introduced in the conversion of the KD Multilingual Global Series dictionaries, i.e, different researchers introduced their own modelling solutions to account for similar notions. Being interoperability a key issue in linked data technologies, building a common space in which these concepts can be agreed on and commonly defined comes as a logical step. The OntoLex community is the natural forum to accomplish this for several reasons:

  1. The extended use of lemon to convert language resources (including lexicographical resources) into linked data,
  2. the nature of the lemon model being descriptive but not prescriptive, which facilities neutrality towards different lexicographic views,
  3. the coming together of the lexicography and the Semantic Web communities, and
  4. the possibility of reusing already available mechanisms in lemon, preventing researchers from "re-inventing the wheel."

Aim and scope

The main goal of this module is to complement the lemon core module, OntoLex, and to overcome its limitations when modelling lexicographic information as linked data in a way that is agnostic to the underlying lexicographic view and minimises information loss.

The scope of the model is two-fold:

  1. Modelling existing lexicographic resources as linked data, to allow their later publication and inter-operation by following Semantic Web standards, and
  2. providing a conceptual / abstract model of language and linguistic objects in lexicography that will allow to build linked data-based lexicographic resources from scratch.
Modelling existing dictionaries as linked data can be done by first extracting the features they have in common and modelling them in a way that is decoupled from specific linguistic views. Although the primary goal might be to model existing resources, the module should remain highly re-usable across dictionaries and useful for new linked data-based lexicographic works built from scratch (see the notion of linked data-native dictionaries at [12]). Naturally, we assume that many specific ingredients from specific dictionaries and views will still need ad-hoc constructs. Still, having a small set of common entities defined in the module will help to overcome the limitations that have been detected in the experiences of instantiating lemon with lexicographic data, and can greatly benefit interoperability.

In terms of applying the module, we propose the following best practice or "rule of thumb" when representing a dictionary as linked data:

  1. As long as the entities in OntoLex and the other lemon modules, together with those of catalogues of linguistic categories (e.g. LexInfo), suffice to represent the information encoded in the lexicographic resource (e.g., lexical entry, part of speech, translation, ...), the OntoLex lexicography module should not be instantiated.
  2. In case that there is some lexicographic information that cannot be modelled by using either OntoLex or any of the other lemon modules (e.g., to denote sense ordering), then the model should be instantiated but avoiding duplicities and keeping extra information to the minimum.
The reason behind this is that this module adds some complexity by providing additional description capabilities to the purely lexical description accounted by OntoLex. If this information is not needed for a specific conversion, i.e, if the lexicographical view is not key, reusing lemon would allow to keep the representation simpler but still sufficient.

Lexicography Module (lexicog)

The following diagram depicts the OntoLex Lexicography module (lexicog). Boxes represent classes of the model. Arrows with filled heads represent object properties. Note that the module provides elements to account for structures common in lexicographic resources (shaded in blue) as well as some to capture annotations and additional information describing lexical elements as found in dictionaries (shaded in orange).

Lexicographic Resource

There are certain situations (see [10]) in which there is no one-to-one mapping between the entries in a dictionary, or lexicographical resource in general, and the lexical entries in a lemon lexicon (lime:Lexicon). For instance, some dictionary entries contain information about their translations and synonyms, but it may be the case that these latter do not have their own dictionary entry in the source dictionary. Still, such translations and synonyms can be treated as "first class citizens" in the graph by representing them as ontolex:LexicalEntr[ies]. In such situations, there is a disparity between the created lime lexicon and the original dictionary, as the linked data representation would include entries not encoded in the original resource as dictionary entries. For this reason, this is class is intended to represent the original lexicographic resource.

Lexicographic Resource (Class)

URI: http://www.w3.org/nl/lemon/lexicog#LexicographicResource

A lexicographic resource represents a collection of lexicographic entries (either ontolex:LexicalEntries or lexicog:SuperEntries) according to the lexicographic criteria followed in the development of the resource.

SubClassOf: void:Dataset, dct:language min 1 xsd:language

A simple example of a lexicographic resource in English that collects several lexical entries is given below. While in this example :person_n and :animal_n are lexical entries compliant to the ontolex:LexicalEntry definition, a dictionary may provide only one single dictionary entry for the word good, compressing its noun, adjective and (colloquial) adverb definitions under one same structure. While the generated lime:Lexicon will contain entries with one single part of speech, following the lemon model, the lexicog:LexicographicResource instance will gather entities as they were presented in the original lexicographic resource in order to keep track of the of original representation in its RDF version:


      :person_n a ontolex:LexicalEntry .
      :animal_n a ontolex:LexicalEntry .
      :good_adj a ontolex:LexicalEntry.
      :good_adv a ontolex:LexicalEntry .
      :good_n a ontolex:LexicalEntry .
      :good_adj_n_adv a lexicog:SuperEntry.
      
      :myDictionary a lexicog:LexicographicResource ;
	       dct:language "en" ;
	       lexicog:entry :person_n, :animal_n ;
               lexicog:superEntry :good_adj_n_adv .

      :myLexicon a lime:Lexicon;
         lime:entry :person_n, :animal_n, :good_n, :good_adj, :good_adv .

Super Entry

The lack of a one-to-one mapping from a dictionary entry to a ontolex:LexicalEntry referred above often translates into a situation in which a structure in the original resource needs to be split into several ontolex:LexicalEntr[ies] for the resulting resource to be OntoLex-compliant. For example, we find this scenario in dictionaries that do not make a dictionary entry split in terms of the part-of-speech of a word. In these resources, the senses of a (polysemous) word are bundled together under the same dictionary entry even though they depend on different parts-of-speech. In these cases, these may be given as annotations just before the sense, e.g. (n.), especially in non-digitally born resources. We can find this configuration also in dictionary entries that take different parts of speech depending on its syntactic behavior (e.g. adjective and adverb) without meaning change.

In addition to bundling together lexical entries with different parts of speech, an entry in a lexicographic resource can also be composed of other (sub-)structures arranged in a particular order or hierarchy, such as senses, nested entries, etc.

lexicog:SuperEntry is introduced to represent the entry as it is encoded in a lexicographic resource, e.g. a dictionary entry, thereby reflecting the arrangement decided upon by lexicographers (supported by the occurrences of the word in the corpus used for dictionary compilation). This class is thus intended to be the counterpart of a dictionary entry in the linked data representation of a dictionary.

Super Entry (Class)

URI: http://www.w3.org/nl/lemon/lexicog#SuperEntry

A super entry is a structural element that represents a lexicographic article or record as it is arranged in a source lexicographic resource (not converted to RDF). As such, it supports the description of lexical entries according to the lexicographic micro-structure, decided upon during a lexicographic resource compilation process. It is a collection of lexicog:SuperEntryComponents, arranged or not in a specific order.

SubClassOf: rdfs:Container

SuperEntryComponent

SuperEntry is intended to fulfill a structural function so as to serve as counterpart of a lexicographic record in a resource, which may or may not be straightly mapped into ontolex:LexicalEntries. However, these lexicographic records often contain sub-structures to refer to senses, sense groups, or nested entries. In an analogy to W3C's CSVW vocabulary , where a csvw:Table consists of csvw:Row(s) and csvw:Column(s), we can state that lexicog:SuperEntry also contains structural components, namely, lexicog:SuperEntryComponents. These components can refer to ontolex:LexicalSenses or ontolex:LexicalEntries, depending on the arrangement in the lexicographic resource. Components can in turn be arranged in a specific order and/or hierarchy, if desired, or just declared to be part of a lexicog:SuperEntry.

Super Entry Component (Class)

URI: http://www.w3.org/nl/lemon/lexicog#SuperEntryComponent

A super entry component is a structural element that represents the (sub-)structures of lexicographic articles providing information about senses or sub-entries. If desired, SuperEntryComponents can be arranged in a specific ordered and/or a hierarchy.

SubClassOf: rdfs:Container

Coming back to the previous example, a lexicographic record for the word animal can group (under the same dictionary entry) descriptions for the lexical entries animal (n.), and animal (adj.), which both share phonetic representation and etymology. The following example is extracted from the American Heritage Dictionary [16]:

an¬∑i¬∑mal

n.

1. Any of numerous multicellular eukaryotic organisms of the kingdom Metazoa (or Animalia) [...]

2. An animal organism other than a human, especially a mammal.

adj.

Here we can detect two different ontolex:LexicalEntries grouped together in the same lexicographic record, each with a sense ordering. If this exact lexicographic record were to be recreated with the lexicog vocabulary, we would have the following RDF triples: 


      :animal_n a ontolex:LexicalEntry . 
      :animal_adj ontolex:LexicalEntry.
      :animal_n_adj a lexicog:SuperEntry.

      :myDictionary a lexicog:LexicographicResource ;
	 dct:language "en" ;
         lexicog:superEntry :animal_n_adj.       
      :myLexicon a lime:Lexicon;
         lime:entry :animal_n, :animal_adj. 
  
      :animal_n_adj a lexicog:SuperEntry ;
         rdfs:member :animal_n_comp; :animal_adj_comp.
           
      :animal_n_comp a lexicog:SuperEntryComponent;
         lexicog:describes :animal_n . 
      :animal_adj_comp a lexicog:SuperEntryComponent;
         lexicog:describes :animal_adj.

Note that this only states that there is a structure (SuperEntry) which contains sub-structures (components) which describe lexical elements, in this case, lexical entries. If a dictionary describes animal (adj.) and animal (n.) in different records or if this exact arrangement is not to be transfered to the RDF representation, this approach would not be necessary.

This same approach can be applied to capture the sense order in the previous example: 


 
      :myDictionary a lexicog:LexicographicResource ;
	 dct:language "en" ;
         lexicog:superEntry :animal_n_adj.       
      :myLexicon a lime:Lexicon;
         lime:entry :animal_n, :animal_adj. 		
		
      :animal_n a ontolex:LexicalEntry ;
         ontolex:sense :animal_n_sense_1, :animal_n_sense_2 . 
         
      :animal_n_sense_1 a ontolex:LexicalSense ;
         ontolex:isLexicalizedSenseOf :animal_n_1_lc . 
      :animal_n_sense_2 a ontolex:LexicalSense;
         ontolex:isLexicalizedSenseOf :animal_n_2_lc .
       
      :animal_n_1_lc  a ontolex:LexicalConcept: 
	skos:definition "Any of numerous multicellular eukaryotic organisms of the kingdom Metazoa (or Animalia)"@en. 
      :animal_n_2_lc  a ontolex:LexicalConcept: 
	skos:definition "An animal organism other than a human, especially a mammal."@en. 
	
      :animal_n_adj a lexicog:SuperEntry;
         rdfs:member :animal_n_comp; :animal_adj_comp.
           
      :animal_n_comp a lexicog:SuperEntryComponent;
	 lexicog:describes :animal_n ; 
         rdf:_1: :animal_n_sense_1_comp; 
	 rdf:_2  :animal_n_sense_2_comp. 

      :animal_n_sense_1_comp a lexicog:SuperEntryComponent ;
	 lexicog:describes :animal_n_sense_1. 
      :animal_n_sense_2_comp a lexicog:SuperEntryComponent ;
	 lexicog:describes :animal_n_sense_2.
